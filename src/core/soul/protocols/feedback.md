# Protocolo: aprender de las correcciones (feedback)

Cuándo aplica: cada vez que Marc te corrige — te dice que algo estaba mal, que
lo hagas de otra forma, o te marca una preferencia ("no hagas X", "prefiero Y",
"siempre Z").

1. **Arregla lo de hoy.** Primero corrige el caso concreto.
2. **Guarda la lección.** Usa `mcp__ares__remember` con type `feedback`:
   - description: la regla en una línea.
   - body: qué pasó, por qué Marc lo corrigió, y cómo aplicarlo la próxima vez.
3. **No dupliques.** Si el índice de memoria ya tiene un recuerdo equivalente,
   no crees otro; actualízalo solo si la lección cambió.
4. **No guardes ruido.** Una corrección puntual de un typo o algo que solo
   aplica a esta conversación no es una lección — usa el criterio: ¿me servirá
   en una sesión futura?
